**ระดับการเรียน**: ประกาศนียบัตรวิชาชีพ (ปวช.) ชั้นปีที่ 1 สาขาเทคโนโลยีสารสนเทศ/คอมพิวเตอร์ธุรกิจ  
**โครงสร้างเวลาเรียน**: 1-3-2 (บรรยาย 1 ชั่วโมง, ปฏิบัติ 3 ชั่วโมง, รวม 4 ชั่วโมงต่อสัปดาห์ | 2 หน่วยกิต)  
**ระยะเวลา**: 18 สัปดาห์ (รวมเวลาสอบกลางภาคและสอบปลายภาค)

## 1. คำอธิบายรายวิชา (Course Description)
ศึกษาและปฏิบัติเกี่ยวกับโครงสร้าง หลักการออกแบบเว็บไซต์ ออกแบบและสร้างเว็บไซต์ด้วยโปรแกรมภาษา HTML5 การตกแต่งข้อความด้วย CSS การแทรกรูปภาพหรือสื่อต่างๆ การสร้างตาราง การสร้างเฟรม การออกแบบและสร้างฟอร์ม การสร้างการเชื่อมโยงในแบบต่างๆ การแทรก JavaScript การทดสอบการทำงานของเว็บไซต์ การอัปโหลดเว็บไซต์ไปยังเว็บเซิร์ฟเวอร์

---

## 2. ผลลัพธ์การเรียนรู้ระดับรายวิชา (Course Learning Outcomes - CLO)
1. **CLO 1**: อธิบายโครงสร้าง ส่วนประกอบ และหลักการทำงานของเว็บไซต์ เว็บโฮสติ้ง และโปรโตคอลการสื่อสารบนอินเทอร์เน็ตได้
2. **CLO 2**: สามารถใช้ภาษา HTML5 เขียนโครงสร้างของเนื้อหาหน้าเว็บ (Semantic Elements, Text, Lists, Tables, Media, Forms) ได้ถูกต้องตามมาตรฐาน
3. **CLO 3**: สามารถใช้ CSS3 ตกแต่งความสวยงามของเว็บเพจ จัดรูปแบบฟอนต์ กำหนดลักษณะ Box Model และจัดหน้าเว็บด้วย Flexbox และ Grid ได้
4. **CLO 4**: สามารถใช้ภาษา JavaScript ขั้นพื้นฐานเพื่อเพิ่มปฏิสัมพันธ์ (Interactivity) หรือฟังก์ชันอย่างง่ายบนหน้าเว็บได้
5. **CLO 5**: มีความรับผิดชอบ รอบคอบ ปลอดภัย ตระหนักถึงลิขสิทธิ์ของสื่อ รูปภาพ โค้ด และจริยธรรมในการเผยแพร่ข้อมูลสู่สาธารณะ

---

## 3. กลยุทธ์การลดบัคในการเขียนโค้ดสำหรับนักเรียนเริ่มต้น (Crucial Coding Strategy)
เพื่อช่วยให้นักเรียน ปวช. 1 ซึ่งเป็นผู้เริ่มต้นเขียนโค้ดครั้งแรก สามารถเรียนรู้ได้อย่างไม่สับสนและลดปัญหาที่พบบ่อย ครูผู้สอนใช้แนวทางต่อไปนี้:
1. **เริ่มต้นจากความเข้าใจธรรมชาติ (Notepad to VS Code)**: ในช่วง 4 สัปดาห์แรก สอนให้พิมพ์โค้ดเองใน Notepad เพื่อเห็นการทำงานของแท็กที่ไม่มีตัวช่วยสะกด จากนั้นย้ายมาใช้ VS Code ในสัปดาห์ที่ 5 เพื่อเพิ่มทักษะและเรียนรู้การใช้งานเครื่องมือแบบมืออาชีพ
2. **เน้นไวยากรณ์มาตรฐาน (Semantic HTML5)**: ปลูกฝังการเปิด-ปิดแท็กให้ครบถ้วน การทำรอยหยักโค้ด (Indentation) และเลี่ยงการใช้แท็กข้อความธรรมดาในการจัดสไตล์ แต่ให้ใช้สไตล์ลิสต์ผ่าน CSS เพื่อความเป็นระเบียบและส่งผลดีต่อ SEO
3. **การเรียนรู้เชิงผลลัพธ์ (Project-Based Learning)**: ทุกสัปดาห์จะมีใบงานปฏิบัติที่เป็นรูปธรรมและสอดคล้องกับหัวข้อ เช่น การสร้างแบบฟอร์มสมัครสมาชิก, การ์ดแสดงราคาสินค้า, หน้าแกลลอรี่ภาพ ซึ่งส่งเสริมให้นักเรียนเห็นประโยชน์ใช้สอยจริงและนำไปรวบรวมเป็น Mini-Project ปลายภาคได้

## 4. แผนการจัดหน่วยเรียนรู้รายสัปดาห์ (18-Week Schedule)

### สัปดาห์ที่ 1: บทนำสู่เว็บไซต์และการสร้างโครงสร้าง HTML เบื้องต้น
*   **ทฤษฎี (1 ชั่วโมง)**:
    *   ความรู้ทั่วไปเกี่ยวกับอินเทอร์เน็ต, เว็บไซต์, หน้าเว็บเพจ, โดเมนเนม, เว็บโฮสติ้ง และ URL
    *   ความแตกต่างระหว่าง Static Website และ Dynamic Website
    *   แนะนำเบื้องต้นเกี่ยวกับภาษาหลัก: HTML (โครงกระดูก), CSS (เสื้อผ้า), JavaScript (พฤติกรรม)
*   **ปฏิบัติ (3 ชั่วโมง) - LAB 1: Basic setup and first webpage**:
    *   การเปิดแสดงผลนามสกุลไฟล์ (File Name Extensions) บน Windows Explorer
    *   การใช้ Text Editor พื้นฐาน (Notepad) สร้างและบันทึกไฟล์เป็นนามสกุล `.html` ในแบบ UTF-8
    *   ทำความคุ้นเคยกับคำสั่งหลัก: `<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`, `<title>`
    *   เขียนแท็กหัวเรื่อง `<h1>`-`<h6>`, ย่อหน้า `<p>`, ตัวตัดบรรทัด `<br/>` และขีดเส้นใต้หัวข้อ `<hr/>`
*   **เครื่องมือหลัก**: Windows Notepad, Chrome/Edge Browser
*   **ฟังก์ชัน/แท็กหลัก**: `<html>`, `<head>`, `<body>`, `<title>`, `<h1>`-`<h6>`, `<p>`, `<br/>`, `<hr/>`
*   **งานปฏิบัติ**: ออกแบบหน้าเว็บเพจ "แนะนำตัวของฉันเบื้องต้น" โดยใช้หัวเรื่อง ย่อหน้า เส้นแบ่ง และลิงก์เบื้องต้น

---

### สัปดาห์ที่ 2: มาตรฐาน Semantic HTML5 และการจัดลำดับข้อความ (Lists)
*   **ทฤษฎี (1 ชั่วโมง)**:
    *   มาตรฐาน HTML5 และความสำคัญของ Semantic Web
    *   ข้อผิดพลาดที่มักพบบ่อย: การใช้แท็กหัวเรื่องผิดประเภทเพื่อจัดขนาดอักษร (Abuse for styling)
    *   ทำความเข้าใจโครงสร้างการแสดงผลกลุ่มข้อความแบบรายการ (Lists)
*   **ปฏิบัติ (3 ชั่วโมง) - LAB 2: Science Worksheet Page**:
    *   การประยุกต์ใช้แท็กจัดรูปแบบอักษร: ตัวหนา `<b>`/`<strong>`, ตัวเอียง `<i>`/`<em>`, ไฮไลต์ `<mark>`, ขีดฆ่า `<del>`, ขีดเส้นใต้ `<ins>`
    *   การใช้ตัวห้อย `<sub>` และตัวยก `<sup>` สำหรับสัญลักษณ์เคมีและคณิตศาสตร์ เช่น \(CO_2\), \(H_2O\), \(x^2\)
    *   เขียนรายการแบบไม่มีลำดับ `<ul>` (Unordered List) และแบบมีลำดับ `<ol>` (Ordered List)
    *   เขียนรายการคำอธิบายศัพท์ `<dl>`, `<dt>`, `<dd>` (Description List)
*   **เครื่องมือหลัก**: Notepad หรือโปรแกรมแก้ไขข้อความธรรมดา
*   **ฟังก์ชัน/แท็กหลัก**: `<strong>`, `<em>`, `<sub>`, `<sup>`, `<ul>`, `<ol>`, `<li>`, `<dl>`, `<dt>`, `<dd>`
*   **งานปฏิบัติ**: พัฒนาหน้าเว็บ "ใบงานวิชาการวิทยาศาสตร์ เรื่อง สารเคมีรอบตัว" โดยใช้ตัวยก ตัวห้อย รายการแบบจุดกลม และพจนานุกรมคำศัพท์ที่จัดหน้าอย่างเป็นระเบียบ

---

### สัปดาห์ที่ 3: สื่อมัลติมีเดีย (Images & Links), ตาราง (Tables) และพื้นฐาน CSS
*   **ทฤษฎี (1 ชั่วโมง)**:
    *   แนวคิดเรื่อง Attributes ใน HTML แท็ก
    *   การจัดการรูปภาพและลิงก์เชื่อมโยงไปยังหน้าเพจอื่นหรือบริการภายนอก
    *   โครงสร้างตารางข้อมูล <table>
    *   เกริ่นนำบทบาทของ CSS (Cascading Style Sheets) และการเชื่อมต่อ 3 รูปแบบ
*   **ปฏิบัติ (3 ชั่วโมง) - LAB 3: Table Design & CSS integration**:
    *   การใส่รูปภาพด้วย `<img>` และแอตทริบิวต์ `src`, `alt`, `width`, `height`, `loading="lazy"`
    *   การสร้างลิงก์ด้วย `<a>` พร้อมใช้ `href` ในการเชื่อมต่อเว็บเพจภายนอก, หน้ารายละเอียดภายใน, หรือลิงก์ส่งอีเมล `mailto:` และเบอร์โทรศัพท์ `tel:`
    *   สร้างตารางด้วย `<table>`, `<tr>`, `<th>`, `<td>` และกำหนดแถวหัวข้อ <thead>, แถวเนื้อหา <tbody>, แถวท้ายตาราง <tfoot>
    *   ทดลองใช้ CSS สไตล์ผ่าน Inline, Internal (ใน `<style>`), และ External (ไฟล์ `.css`)
*   **เครื่องมือหลัก**: Notepad หรือโปรแกรมแก้ไขข้อความธรรมดา, CSS พื้นฐาน
*   **ฟังก์ชัน/แท็กหลัก**: `<img>`, `<a>`, `<table>`, `<thead>`, `<tbody>`, `<tfoot>`, `<style>`
*   **งานปฏิบัติ**: สร้างหน้าเว็บ "ตารางบันทึกรายรับ-รายจ่ายประจำวัน" ที่มีภาพประกอบตกแต่ง ใส่ลิงก์เชื่อมโยงไปยังบัญชีผู้ให้บริการโอนเงิน และกำหนดสีสันเส้นขอบตารางด้วย CSS เบื้องต้น

---

### สัปดาห์ที่ 4: รูปแบบตารางขั้นสูง (colspan/rowspan) และแบบฟอร์ม (Forms)
*   **ทฤษฎี (1 ชั่วโมง)**:
    *   การควบรวมแถวและคอลัมน์ในตาราง (Merge Cells) ด้วย colspan และ rowspan
    *   หลักการรับข้อมูลจากผู้ใช้ผ่านฟอร์ม HTML (Form processing concept)
    *   ความแตกต่างระหว่างการส่งข้อมูลผ่าน Method GET และ POST
*   **ปฏิบัติ (3 ชั่วโมง) - LAB 4: User Registration Form**:
    *   สร้างโครงสร้างตารางวิเคราะห์เกรดที่มีการรวมคอลัมน์ (`colspan`) และรวมแถว (`rowspan`)
    *   ประกาศฟอร์มด้วยแท็ก `<form>`
    *   ใช้งานชุดป้อนข้อมูล `<input>` ประเภทต่างๆ: text, password, email, number, date, radio (เลือกเพศ), checkbox (ความสนใจ)
    *   สร้างเมนูเลือกแบบ Dropdown ด้วย `<select>` และ `<option>`
    *   สร้างกล่องข้อความบรรยายขนาดใหญ่ด้วย `<textarea>`
    *   จัดกลุ่มหน้าต่างกรอกข้อมูลด้วย `<fieldset>` และคำอธิบายกลุ่ม `<legend>`
    *   สร้างปุ่มกดด้วย `<button>` (submit, reset)
*   **เครื่องมือหลัก**: Notepad หรือโปรแกรมแก้ไขข้อความธรรมดา
*   **ฟังก์ชัน/แท็กหลัก**: `colspan`, `rowspan`, `<form>`, `<input>`, `<select>`, `<textarea>`, `<fieldset>`, `<legend>`
*   **งานปฏิบัติ**: พัฒนาหน้าเว็บ "แบบฟอร์มสมัครสมาชิกใหม่ (Register Page)" ที่ออกแบบโครงสร้างเรียบร้อย จัดวางฟอร์มและปุ่มส่งข้อมูลอย่างเหมาะสม

### สัปดาห์ที่ 5: สภาพแวดล้อม VS Code, สื่อมัลติมีเดีย HTML5 ขั้นสูง และ CSS Selectors
*   **ทฤษฎี (1 ชั่วโมง)**:
    *   แนะนำเหตุผลในการย้ายไปใช้ IDE มืออาชีพอย่าง Visual Studio Code
    *   การติดตั้งและการตั้งค่า Extension ที่จำเป็น (HTML5 Boilerplate, Live Server, Prettier)
    *   ความเข้าใจเรื่อง Block Element vs Inline Element และโครงสร้าง Div/Span
    *   การเข้าถึงส่วนประกอบเพื่อกำหนดสไตล์ด้วย CSS Selectors
*   **ปฏิบัติ (3 ชั่วโมง) - LAB 5: VS Code setup and Selector mapping**:
    *   ติดตั้งและฝึกคีย์ลัดพื้นฐานใน VS Code (Ctrl+P, Ctrl+B, Ctrl+/, Shift+Alt+F)
    *   ใช้งานแท็กมัลติมีเดีย HTML5: `<video>` และ `<audio>` (แอตทริบิวต์: controls, poster, loop, muted)
    *   ทดลองใช้กล่องแบ่งสัดส่วนเลย์เอาต์ `<div>` และจัดกลุ่มอักษรในบรรทัด `<span>`
    *   ทดสอบความแตกต่างของ CSS Selectors: Element Selector, Class Selector (.), ID Selector (#)
*   **เครื่องมือหลัก**: Visual Studio Code, Live Server, HTML5 Media Elements
*   **ฟังก์ชัน/แท็กหลัก**: `<video>`, `<audio>`, `<div>`, `<span>`, CSS Class & ID
*   **งานปฏิบัติ**: ออกแบบหน้าเพจ "แนะนำโฆษณาสินค้า/วิดีโอเพลง" โดยแสดงไฟล์วิดีโอ/เสียงที่มีปุ่มควบคุม และใช้ CSS คลาสในการจัดแต่งลักษณะตัวอักษรของกล่อง `div` แยกตามแบรนด์สินค้า

### สัปดาห์ที่ 6: CSS Typography และแบบจำลองกล่อง (CSS Box Model)
*   **ทฤษฎี (1 ชั่วโมง)**:
    *   การออกแบบตัวอักษรบนหน้าเว็บ (Typography Design)
    *   แอตทริบิวต์การแสดงผลตัวอักษร: การใช้ฟอนต์ตระกูลสำรอง (Font fallbacks), ขนาด (px, rem), และการทำเงาข้อความ
    *   ทำความเข้าใจกฎทางเลย์เอาต์หลักของเว็บ: CSS Box Model (Content, Padding, Border, Margin)
*   **ปฏิบัติ (3 ชั่วโมง) - LAB 6: Product Card Design**:
    *   ตั้งค่าฟอนต์อักษรด้วย `font-family`, `font-size`, `font-weight`, `text-decoration` (none, line-through)
    *   ใช้คุณสมบัติการตัดทอนคำเมื่อตัวอักษรล้นกล่อง (`white-space: nowrap`, `overflow: hidden`, `text-overflow: ellipsis`)
    *   ทดสอบสไตล์ของ Box Model: กำหนดขนาด (`width`/`height`), ระยะห่างภายใน (`padding`), เส้นขอบกล่อง (`border`), ระยะห่างภายนอก (`margin`)
    *   การปรับปรุงความสวยงามของกล่องด้วย `border-radius` (ทำรูปทรงกลมหรือเหลี่ยมมุมมน) และ `box-shadow`
*   **เครื่องมือหลัก**: VS Code, CSS Stylesheet
*   **ฟังก์ชัน/แท็กหลัก**: `font-family`, `text-overflow: ellipsis`, `padding`, `border`, `margin`, `border-radius`, `box-shadow`
*   **งานปฏิบัติ**: สร้างหน้าเว็บ "การ์ดขายรองเท้าพรีเมียม (Premium Sneakers Card)" ที่มีการนำเสนอรูปสินค้า ชื่อหัวข้อ รายละเอียดที่ตัดด้วยจุดไข่ปลาหากยาวเกินไป ราคาปกติและราคาลดพิเศษขีดฆ่า และมีปุ่มซื้อที่เปลี่ยนสีเมื่อวางเมาส์ทับ

### สัปดาห์ที่ 7: การจัดหน้าเว็บเพจยืดหยุ่นด้วย CSS Flexbox
*   **ทฤษฎี (1 ชั่วโมง)**:
    *   ปัญหาและข้อจำกัดของเลย์เอาต์หน้าเว็บแบบดั้งเดิม (Table Layout, Float Layout)
    *   คอนเซปต์ระบบ CSS Flexbox: Flex Container และ Flex Items
    *   การกำหนดแกนหลัก (Main Axis) และแกนรอง (Cross Axis)
*   **ปฏิบัติ (3 ชั่วโมง) - LAB 7: Navigation Bar & Multi-column Cards**:
    *   เริ่มต้นใช้งานด้วย `display: flex`
    *   ควบคุมทิศทางการเรียงด้วย `flex-direction` (row, column) และการเว้นระยะด้วย `gap`
    *   จัดตำแหน่งแกนหลักด้วย `justify-content` (flex-start, center, space-between, space-around)
    *   จัดตำแหน่งแกนรองด้วย `align-items` (flex-start, center, stretch)
    *   การควบคุมการขึ้นบรรทัดใหม่ด้วย `flex-wrap: wrap`
*   **เครื่องมือหลัก**: VS Code, CSS Flexbox
*   **ฟังก์ชัน/แท็กหลัก**: `display: flex`, `flex-direction`, `justify-content`, `align-items`, `gap`, `flex-wrap`
*   **งานปฏิบัติ**: พัฒนาหน้าเว็บ "แถบนำทางส่วนหัวเว็บไซต์ (Header Navigation)" และโครงสร้าง "รายการสินค้าแบบการ์ด 3 คอลัมน์" ที่เปลี่ยนรูปเลย์เอาต์เป็นคอลัมน์เดี่ยวอัตโนมัติบนขนาดหน้าจอที่แคบลง

### สัปดาห์ที่ 8: ทบทวนเตรียมสอบกลางภาค (Midterm Review Project)

    *   วิเคราะห์ข้อบกพร่องทั่วไปในโค้ดและวิธีการดีบักโครงสร้างหน้าเว็บผ่าน Chrome DevTools (Inspect element)
*   **ปฏิบัติ (3 ชั่วโมง) - LAB 8: Comprehensive Midterm Project**:

    *   ต้องประกอบด้วย: โครงสร้างประวัติ (Semantic Tags), ตารางแสดงประวัติการศึกษา (Tables with colspan/rowspan), วิดีโอแนะนำตัว (Video tag), ฟอร์มติดต่อกลับ (Forms), และจัดรูปแบบด้วย CSS Box Model + Flexbox

### สัปดาห์ที่ 10: การจัดโครงสร้างเลย์เอาต์ขั้นสูงด้วย CSS Grid Layout
*   **ทฤษฎี (1 ชั่วโมง)**:
    *   แนวคิดการจัดเลย์เอาต์แบบ 2 มิติ (แถวและคอลัมน์พร้อมกัน)
    *   ความแตกต่างระหว่าง Flexbox (1 มิติ) และ Grid (2 มิติ)
    *   แนวคิด Grid Container และ Grid Items
*   **ปฏิบัติ (3 ชั่วโมง) - LAB 10: Gallery Grid Layout**:
    *   เริ่มต้นใช้งานด้วย `display: grid`
    *   กำหนดสัดส่วนช่องตารางด้วย `grid-template-columns` (การใช้หน่วย `fr`, `repeat()`, `auto-fit`)
    *   การกำหนดแถวด้วย `grid-template-rows` และระยะห่างด้วย `gap`
    *   การควบคุมพื้นที่สิ่งของด้วย `grid-column` และ `grid-row` (การใช้ `span`)
*   **เครื่องมือหลัก**: VS Code, CSS Grid
*   **งานปฏิบัติ**: ออกแบบหน้าแกลลอรีรูปภาพ (Image Gallery Dashboard) ที่วางภาพในขนาดเล็กและใหญ่สลับกันอย่างสวยงามด้วยระบบกริด

### สัปดาห์ที่ 11: การออกแบบเว็บเพจที่ยืดหยุ่นรองรับทุกหน้าจอ (Responsive Web Design)
*   **ทฤษฎี (1 ชั่วโมง)**:
    *   หลักการออกแบบ Responsive Web Design และแนวคิด Mobile-First Design
    *   การทำงานของ Viewport Meta tag

*   **ปฏิบัติ (3 ชั่วโมง) - LAB 11: Responsive Web Page**:

    *   ปรับเลย์เอาต์หน้าต่างของ Grid และ Flexbox ให้ปรับเปลี่ยนตามขนาดจอ Desktop, Tablet, และ Mobile
    *   การใช้หน่วยวัดความยืดหยุ่น เช่น %, vw, vh, rem, em
*   **เครื่องมือหลัก**: VS Code, Responsive Viewport
*   **งานปฏิบัติ**: ปรับปรุงการ์ดสินค้าและตารางข้อมูลจากบทเรียนที่ผ่านมาให้แสดงผลได้สวยงามบนอุปกรณ์สมาร์ทโฟนโดยไม่มีเนื้อหาหลุดขอบหน้าจอ

### สัปดาห์ที่ 12: ลูกเล่นความเคลื่อนไหว (CSS Transitions & Animations)
*   **ทฤษฎี (1 ชั่วโมง)**:
    *   การสร้างปฏิสัมพันธ์กับผู้ใช้เชิงภาพ (Visual Feedback)
    *   ทฤษฎีและคุณสมบัติของ Transition (การเปลี่ยนสถานะจากจุดหนึ่งไปอีกจุดอย่างสมูท)
    *   การสร้างแอนิเมชั่นขั้นสูงโดยใช้ Keyframes
*   **ปฏิบัติ (3 ชั่วโมง) - LAB 12: Interactive Hover Cards**:
    *   ใช้งานสไตล์ `transition`: transition-property, transition-duration, transition-timing-function
    *   สร้างการเคลื่อนไหวผ่านตัวกรองทรานส์ฟอร์ม: `transform` (scale, rotate, translate)
    *   เขียน CSS `@keyframes` เพื่อทำเอฟเฟกต์หมุน วน หรือสั่นไหวของปุ่ม
*   **เครื่องมือหลัก**: VS Code, CSS Animation
*   **งานปฏิบัติ**: สร้างหน้าเว็บร้านค้าออนไลน์ที่มีรูปภาพซูมเข้าเบาๆ และปุ่มซื้อสินค้ากะพริบหรือสั่นเพื่อเรียกความสนใจเมื่อผู้ใช้เลื่อนหน้าจอมาเจอ (Hover & Keyframes)

### สัปดาห์ที่ 13: แนะนำเครื่องมือทุ่นแรงด้วย CSS Frameworks (Tailwind CSS)

*   **ปฏิบัติ (3 ชั่วโมง) - LAB 13: Rapid Prototyping with Tailwind CSS**:
    *   เชื่อมโยงไฟล์สเปก CDN ของ Tailwind CSS
    *   เขียนโครงสร้างกล่อง ข้อความ ปุ่ม และเลย์เอาต์ Flex/Grid โดยใช้ Utility Classes
    *   เปรียบเทียบความเร็วและความเป็นระบบเมื่อเทียบกับการเขียน CSS ไฟล์แยกแบบเดิม
*   **เครื่องมือหลัก**: VS Code, Tailwind CSS CDN
*   **งานปฏิบัติ**: ออกแบบหน้าโปรโมตงานนิทรรศการ (Landing Page) ที่สวยงาม รวดเร็ว ด้วยคลาสของ Tailwind CSS

### สัปดาห์ที่ 14: ความรู้เบื้องต้นและการใช้งานภาษา JavaScript
*   **ทฤษฎี (1 ชั่วโมง)**:
    *   บทบาทของ JavaScript (JS) ในการทำให้หน้าเว็บมีปฏิสัมพันธ์ (Interactivity)
    *   โครงสร้างประโยคไวยากรณ์ ตัวแปร (let, const) และประเภทข้อมูล
    *   โครงสร้างควบคุมเงื่อนไข (if-else) และฟังก์ชัน (Functions)
*   **ปฏิบัติ (3 ชั่วโมง) - LAB 14: Basic Scripting**:
    *   การแทรกสคริปต์ผ่านแท็ก `<script>` ทั้งภายในและเขียนไฟล์แยก `.js`
    *   ทดลองใช้คำสั่งแสดงผล: `console.log()`, `alert()`, `prompt()`
    *   เขียนฟังก์ชันคำนวณคณิตศาสตร์เบื้องต้น (บวก ลบ คูณ หาร) รับค่าจากฟอร์มและคำนวณผล
*   **เครื่องมือหลัก**: VS Code, JavaScript Engine
*   **งานปฏิบัติ**: เขียนโปรแกรมสคริปต์แสดงกล่องข้อความต้อนรับตามชื่อที่ผู้ใช้พิมพ์กรอกเข้ามา และแจ้งเตือนหากไม่ได้กรอกฟิลด์ข้อมูลสำคัญ

### สัปดาห์ที่ 15: การจัดการโครงสร้าง DOM และสร้างฟังก์ชันตอบสนองผู้ใช้ด้วย JavaScript
*   **ทฤษฎี (1 ชั่วโมง)**:
    *   ทำความเข้าใจโมเดลโครงสร้างหน้าเว็บ DOM (Document Object Model)
    *   การเลือกออบเจกต์ในหน้าเว็บด้วย JS (querySelector, getElementById)
    *   แนวคิดการจับเหตุการณ์ (Event Listeners) เช่น การคลิกเมาส์ การเลื่อน การกดปุ่มพิมพ์
*   **ปฏิบัติ (3 ชั่วโมง) - LAB 15: Dynamic Web Control**:
    *   การใช้ `document.querySelector()` เข้าถึงแท็กต่างๆ
    *   การแก้ไขเนื้อหาข้อความ (`textContent`, `innerHTML`) และปรับสีสันสไตล์ผ่าน JavaScript (`element.style`)
    *   สร้างระบบสลับสีพื้นหลังหน้าเว็บเป็นโหมดมืด/สว่าง (Dark/Light Mode Toggle)
    *   เขียนโค้ดตรวจสอบความถูกต้องของฟอร์มเบื้องต้น (Simple Form Validation) ก่อนส่งข้อมูล
*   **เครื่องมือหลัก**: VS Code, DOM API
*   **งานปฏิบัติ**: ออกแบบหน้าต่างสลับโหมดหน้าเว็บส่วนตัว (Theme switcher) และทำแบบฟอร์มตรวจสอบความก้าวหน้าการกรอกข้อมูลอีเมลและรหัสผ่าน หากสั้นเกินไปให้แสดงตัวอักษรสีแดงเตือนทันทีโดยไม่ต้องรีโหลดหน้าเพจ

### สัปดาห์ที่ 16: การวางแผนและการสร้างโครงลวดเว็บไซต์ (Mini-Project Wireframing)

    *   สร้างแผนผังของไซต์อย่างน้อย 3 หน้าเพจ (Home, About, Contact) และวาดแบบร่างเลย์เอาต์ (Wireframe) ลงกระดาษหรือโปรแกรม Figma
    *   สร้าง Repository บน GitHub และสร้างไฟล์โครงสร้างหลักรอไว้
*   **เครื่องมือหลัก**: Figma, Git/GitHub, Sketch tools
*   **งานปฏิบัติ**: ส่งมอบแผนผังเว็บไซต์ ภาพร่างแบบโครงสร้าง และโครงร่างโฟลเดอร์ไฟล์บน GitHub ให้ครูผู้สอนตรวจประเมินเบื้องต้น

*   **ทฤษฎี (1 ชั่วโมง)**:

    *   ปรับขนาดภาพและปรับแต่งโค้ดให้โหลดเร็วขึ้น

*   **เครื่องมือหลัก**: Git, GitHub Pages / Netlify hosting
*   **งานปฏิบัติ**: อัปเดตไฟล์ README.md บน GitHub ให้มีคำอธิบายตัวโครงงานและแนบลิงก์หน้าเว็บไซต์จริงที่ออนไลน์เรียบร้อยแล้วเพื่อให้บุคคลทั่วไปเข้าเยี่ยมชมได้
