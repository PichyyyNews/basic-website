## Slide 2: วัตถุประสงค์การเรียนรู้
*   เข้าใจกฎกติกาการใช้ Semantic HTML5 เพื่อจัดโครงสร้างข้อความ
*   รู้จักความแตกต่างระหว่างการเขียนโค้ดที่ถูกต้อง (Semantic) กับการเขียนโค้ดที่ไม่แนะนำ (Abuse for styling)
*   เลือกใช้แท็กจัดรูปแบบอักษรระดับเจาะจง (Subscript, Superscript, Emphasis) ได้เหมาะสม
*   จัดระเบียบเนื้อหาด้วยรายการชนิดต่าง ๆ (Unordered, Ordered, Description Lists) ได้

## Slide 3: อะไรคือ Semantic HTML?
*   **Semantic HTML:** การเลือกใช้แท็กที่มีความหมายในตนเองเพื่อระบุบทบาทของข้อความนั้นจริง ๆ 
*   **ไม่ใช่เพียงเพื่อแต่งหน้าตา:** 
    *   ❌ ใช้ `<h1>` เพียงเพราะอยากให้ตัวหนังสือตัวหนาและมีขนาดใหญ่ (Abuse for styling)
    *   ✅ ใช้ `<h1>` เพื่อบ่งบอกว่านี่คือ "หัวข้อเรื่องใหญ่ที่สุดประจำหน้าเพจ" และใช้ CSS ปรับขนาดแทนหากต้องการความแตกต่าง

## Slide 4: แท็กจัดสไตล์อักษรเฉพาะจุด (Text Formatting)
*   ตัวหนา: `<b>` (Bold) และ `<strong>` (สำคัญ)
*   ตัวเอียง: `<i>` (Italic) และ `<em>` (เน้นความหมาย)
*   ขีดไฮไลต์: `<mark>` (ไฮไลต์ข้อความ)
*   ข้อความขนาดเล็ก: `<small>`
*   ขีดฆ่า/ขีดเส้นใต้: `<del>` (ลบ) และ `<ins>` (เพิ่มใหม่)

## Slide 5: ตัวห้อยและตัวยก (Subscript & Superscript)
*   **Subscript `<sub>`:** ตัวอักษรห้อยลงด้านล่าง
    *   *ตัวอย่าง:* สัญลักษณ์ทางเคมี เช่น น้ำ `H<sub>2</sub>O` และ ก๊าซคาร์บอนไดออกไซด์ `CO<sub>2</sub>`
*   **Superscript `<sup>`:** ตัวอักษรยกขึ้นด้านบน
    *   *ตัวอย่าง:* สัญลักษณ์ทางคณิตศาสตร์ เช่น เลขยกกำลัง `x<sup>2</sup>` หรือ สัญลักษณ์ประจุไฟฟ้า `SO<sub>4</sub><sup>2-</sup>`

## Slide 6: ลิสต์รายการ HTML Lists
*   มีไว้สำหรับรวบรวมรายการข้อมูลที่สอดคล้องกันเป็นข้อๆ
*   ช่วยให้หน้าเว็บมีความสะอาด เป็นสัดส่วน และเพิ่มความเป็นระเบียบ
*   มี 3 ชนิดย่อยตามลักษณะหน้าที่การใช้งาน

## Slide 7: Unordered List `<ul>` (รายการไม่มีลำดับ)
*   ใช้สัญลักษณ์จุดดำกลม (Bullet points) นำหน้าข้อมูล
*   มีรูปแบบโครงสร้างคู่กับแท็ก `<li>` (List Item) ด้านในเสมอ
*   *ตัวอย่าง:*
    ```html
    <ul>
        <li>วิชา HTML</li>
        <li>วิชา CSS</li>
    </ul>
    ```

## Slide 8: Ordered List `<ol>` (รายการมีลำดับ)
*   ใช้ตัวเลขหรือตัวอักษรนำหน้าเรียงกันตามลำดับ (1, 2, 3...)
*   มีประโยชน์สำหรับการเขียนอธิบายขั้นตอนการทำงาน
*   *ตัวอย่าง:*
    ```html
    <ol>
        <li>ขั้นตอนเตรียมไฟล์</li>
        <li>ขั้นตอนพิมพ์โค้ด</li>
    </ol>
    ```

## Slide 9: Description List `<dl>` (รายการคำอธิบายศัพท์)
*   เหมาะสำหรับการทำพจนานุกรมศัพท์หรือแจกแจงนิยามคำศัพท์
*   ประกอบด้วย 3 แท็กทำงานร่วมกัน:
    *   `<dl>`: ขอบเขตของรายการ
    *   `<dt>` (Description Term): หัวข้อคำศัพท์ที่ต้องการอธิบาย
    *   `<dd>` (Description Data): ข้อมูลรายละเอียดคำนิยามคำศัพท์ (จะมีการเยื้องขวาอัตโนมัติ)
